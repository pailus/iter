# Laravel Exploitation & Pentesting

Laravel is the most popular PHP framework. Misconfigurations in Laravel often lead to complete application compromise, especially when environment variables or debug modes are exposed.

## 1. Debug Mode & Environment Leaks

If `APP_DEBUG=true` in the `.env` file, Laravel will render an Ignition error page containing sensitive environment variables whenever an exception occurs.
*   **Attack:** Force an error (e.g., by malforming a URL parameter or sending an invalid POST body) and search the stack trace for `APP_KEY`, `DB_PASSWORD`, and `REDIS_PASSWORD`.
*   **Environment Exposure:** Always explicitly check common paths for the environment file:
    *   `/.env`
    *   `/.env.example`
    *   `/.env.backup`

## 2. Abusing the APP_KEY

The `APP_KEY` is central to Laravel's security (used for encrypting cookies, sessions, and CSRF tokens). If you obtain the `APP_KEY`:
1.  **Session Forgery:** You can decrypt and modify the Laravel session cookie to impersonate other users (like `user_id = 1`).
2.  **RCE via Deserialization (Older Versions):** In Laravel <= 5.6.29, if you have the `APP_KEY`, you can encrypt a malicious PHP payload (usually generated by `phpggc`) and pass it via the `X-XSRF-TOKEN` cookie or as a serialized job to achieve Remote Code Execution.

## 3. Storage & Log Files

By default, Laravel writes structured logs to the `/storage/logs/` directory.
*   **Target:** `storage/logs/laravel.log` (Check if exposed publicly).
*   **Impact:** Since exceptions dump session data and query bindings, the active log file frequently leaks password reset tokens, session IDs, and database queries.

## 4. Unrestricted File Uploads in Storage

When testing upload endpoints, developers often use Laravel's `$request->file('avatar')->store('public')`. 
If you can upload a `.php` file, check if the `storage/app/public` directory has been linked to the web root via `public/storage/`. If so, your PHP shell will execute when accessed at `/storage/avatar.php`.
